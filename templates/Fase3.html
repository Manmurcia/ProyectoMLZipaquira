Fase 3: Ingeniería del Modelo - Proyecto CRISP-ML

Proyecto

Clasificación de Riesgo de Enfermedades Respiratorias en Adultos Mayores de Zipaquirá

En esta fase se selecciona, entrena, ajusta y valida el modelo de aprendizaje automático que clasificará a los adultos mayores según su riesgo de desarrollar enfermedades respiratorias.

1. Selección del Modelo

Se trata de un problema de clasificación multiclase (riesgo: Bajo, Medio o Alto). Se evaluarán los siguientes modelos:

  • Random Forest: Alta precisión y tolerancia al ruido.
  • XGBoost: Potente y eficiente.
  • Regresión logística multiclase: Modelo base inicial.
  • Árbol de decisión: Interpretabilidad sencilla.

Se seleccionará el mejor en función del rendimiento y aplicabilidad.
Para este caso se usara Random Forest por su biuen rendimiento en poblemas de clasificacion con datos tabulares, no necesita normalizar los datos y proporciona una metrica de importancia de variables util para interpretar

2. Entrenamiento del Modelo

Los datos se dividirán en conjunto de entrenamiento (80%) y prueba (20%) usando división estratificada para mantener el equilibrio de clases. Luego, el modelo será entrenado usando los datos preparados en la fase 2.

3. Ajuste de Hiperparámetros

Se utilizará búsqueda en malla (Grid Search) para optimizar hiperparámetros como:

  • Número de árboles (n_estimators)
  • Profundidad máxima (max_depth)
  • Tamaño mínimo de división (min_samples_split)

Esto permitirá mejorar el rendimiento del modelo sin sobreajustar.

4. Validación del Modelo

Se evaluará el desempeño del modelo usando métricas como:

  • Matriz de confusión
  • Precisión
  • Recall
  • F1-score

Estas métricas permitirán identificar qué tan bien clasifica cada nivel de riesgo.

5. Interpretación del Modelo

Se analizará la importancia de las características predictoras para entender qué variables tienen mayor peso en la clasificación del riesgo respiratorio. Esto facilita la toma de decisiones por parte del personal de salud.

Resultado Esperado

Al finalizar esta fase se tendrá un modelo entrenado, ajustado y validado, listo para ser implementado en un entorno de prueba o producción.
